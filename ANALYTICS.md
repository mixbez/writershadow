# WriterShadow Analytics

Аналитика WriterShadow позволяет просматривать статистику написания ваших авторов через веб-интерфейс.

## 🌐 Как открыть аналитику в браузере

### Требования
- Backend WriterShadow должен быть запущен
- Сервис доступен на `http://localhost:3001` (или на вашем URL сервера)

### Адрес аналитики
```
http://localhost:3001/analytics.html
```

Или если сервер находится на удаленном хосте:
```
https://ваш-домен.com/analytics
```

## 📊 Что доступно в аналитике

### Главные метрики
- **Всего знаков** — сумма всех написанных символов
- **Черновиков** — сумма всех сохраненных черновиков
- **Постов опубликовано** — сумма всех опубликованных постов
- **Активных пользователей** — количество пользователей с активностью

### Таблица статистики по пользователям
Показывает для каждого пользователя:
- Имя пользователя (username)
- Всего знаков написано
- Количество черновиков
- Количество опубликованных постов
- Активных дней (дней, в которые было написано что-либо)

## 🎯 Фильтрация по датам

На странице аналитики вы можете:

1. **Выбрать произвольный период**:
   - Нажмите на поле "От" и выберите начальную дату
   - Нажмите на поле "До" и выберите конечную дату
   - Нажмите кнопку "Обновить"

2. **Использовать быстрые фильтры**:
   - **Сегодня** — показывает статистику текущего дня
   - **Последняя неделя** — последние 7 дней
   - **Последний месяц** — последние 30 дней

## 🔌 API Endpoints

Если вам нужны данные в формате JSON для интеграции:

### GET `/api/analytics`

**Параметры:**
- `from` — начальная дата (YYYY-MM-DD)
- `to` — конечная дата (YYYY-MM-DD)

**Пример:**
```bash
curl "http://localhost:3001/api/analytics?from=2025-01-01&to=2025-01-31"
```

## 📈 Как данные собираются

### Отслеживание активности
Система автоматически отслеживает:

1. **Черновики** — каждое сообщение в группе черновиков
   - Сохраняется текст и количество символов
   - Дата и время написания
   - Связь с пользователем

2. **Посты** — публикации в канале
   - Сохраняется собранный текст
   - Дата публикации
   - Статус (draft/published)

3. **Суточная статистика** (`daily_stats`)
   - Агрегированные данные по дням
   - Количество написанных символов
   - Количество черновиков
   - Количество опубликованных постов

### Таблица базы данных

```sql
CREATE TABLE daily_stats (
  id              SERIAL PRIMARY KEY,
  user_id         INTEGER NOT NULL,
  date            DATE NOT NULL,
  chars_written   INTEGER DEFAULT 0,
  drafts_count    INTEGER DEFAULT 0,
  posts_published INTEGER DEFAULT 0,
  UNIQUE(user_id, date)
);
```

## 🔐 Безопасность

### Текущая конфигурация
- Аналитика доступна **без аутентификации** (локально)
- На боевом сервере рекомендуется добавить защиту через:
  - Reverse proxy (Caddy/Nginx) с Basic Auth
  - Firewall правила
  - API ключи (если нужна интеграция с внешними сервисами)

### Рекомендации для боевого сервера

**Вариант 1: Basic Auth через Caddy**
```
reverse_proxy /analytics* localhost:3001 {
  header_up Authorization "Basic dXNlcm5hbWU6cGFzc3dvcmQ="
}
```

**Вариант 2: Firewall**
```bash
# Разрешить доступ только с определенного IP
iptables -A INPUT -p tcp --dport 3001 -s 192.168.1.1 -j ACCEPT
```

## 🐛 Troubleshooting

### Страница не загружается
1. Проверьте, запущен ли сервис
2. Убедитесь, что порт 3001 доступен
3. Проверьте логи сервера

### Нет данных в таблице
1. Проверьте, есть ли данные в базе: `SELECT COUNT(*) FROM daily_stats;`
2. Убедитесь, что пользователи уже отправляли сообщения в группу черновиков
3. Проверьте выбранный период дат

### API возвращает ошибку 500
Проверьте логи сервера и убедитесь, что:
- Параметры `from` и `to` имеют формат YYYY-MM-DD
- База данных доступна
- Таблица `daily_stats` существует

## 📞 Поддержка

Если возникли проблемы:

2. Убедитесь, что все переменные окружения установлены
